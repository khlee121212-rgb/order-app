# 프런트엔드 UI PRD - 주문하기 화면

## 1. 개요

### 1.1 화면명
주문하기 화면 (Order Screen)

### 1.2 화면 목적
사용자가 커피 메뉴를 둘러보고, 옵션을 선택해 장바구니에 담은 뒤 주문을 완료할 수 있는 고객용 화면

### 1.3 사용자
- 카페를 방문한 고객 (별도의 로그인/인증 없이 누구나 이용)

## 2. 화면 구성

주문하기 화면은 위에서부터 아래로 다음 세 영역으로 구성된다.

1. 상단 헤더 (Header)
2. 메뉴 목록 (Menu List)
3. 장바구니 (Cart)

```
┌─────────────────────────────────────────────┐
│  COZY                      [주문하기]  관리자  │  ← 헤더
├─────────────────────────────────────────────┤
│  ┌────────┐  ┌────────┐  ┌────────┐          │
│  │ 메뉴1   │  │ 메뉴2   │  │ 메뉴3   │   ...     │  ← 메뉴 목록
│  └────────┘  └────────┘  └────────┘          │
├─────────────────────────────────────────────┤
│  장바구니                                      │  ← 장바구니
│   - 항목 목록 / 총 금액 / 주문하기 버튼          │
└─────────────────────────────────────────────┘
```

## 3. 영역별 상세 요구사항

### 3.1 상단 헤더 (Header)

| 항목 | 설명 |
| --- | --- |
| 로고 | 좌측에 서비스명 `COZY` 텍스트 표시 |
| 내비게이션 | 우측에 `주문하기`, `관리자` 두 개의 메뉴 표시 |
| 활성 상태 | 현재 화면인 `주문하기`가 선택(강조) 상태로 표시 |
| 이동 | `관리자` 클릭 시 관리자 화면으로 이동 |

- 헤더는 화면 상단에 고정되며 모든 영역에서 항상 보인다.

### 3.2 메뉴 목록 (Menu List)

- 카드 형태의 메뉴 아이템을 가로로 나열한다. (와이어프레임 기준 한 줄 3개)
- 화면 너비가 좁아지면 카드가 자연스럽게 줄바꿈되어 배치된다. (반응형)
- 메뉴 데이터는 백엔드 API에서 조회한다.

각 메뉴 카드의 구성 요소:

| 요소 | 설명 |
| --- | --- |
| 메뉴 이미지 | 메뉴 대표 이미지 (이미지가 없을 경우 placeholder 표시) |
| 메뉴 이름 | 예) `아메리카노(ICE)` |
| 가격 | 예) `4,000원` (천 단위 콤마 표기) |
| 메뉴 설명 | 메뉴에 대한 간단한 설명 텍스트 |
| 옵션 선택 | 체크박스 형태의 옵션 목록 (예: `샷 추가 (+500원)`, `시럽 추가 (+0원)`) |
| 담기 버튼 | 선택한 옵션과 함께 해당 메뉴를 장바구니에 담는 버튼 |

옵션 동작:
- 각 옵션은 체크박스로 다중 선택 가능하다.
- 옵션명 옆에 추가 금액을 `(+금액원)` 형태로 표시한다.
- 추가 금액이 0원인 옵션도 `(+0원)`으로 표기한다.

담기 버튼 동작:
- `담기` 클릭 시 현재 카드에서 선택된 옵션과 함께 해당 메뉴 1개가 장바구니에 추가된다.
- 동일한 메뉴 + 동일한 옵션 조합을 다시 담으면 장바구니에서 수량이 증가한다.
- 동일한 메뉴라도 옵션 조합이 다르면 장바구니에 별도 항목으로 추가된다.

### 3.3 장바구니 (Cart)

- 메뉴 목록 하단에 박스 형태로 표시된다.
- 영역 상단에 `장바구니` 제목을 표시한다.

장바구니 항목(라인) 구성:

| 요소 | 설명 |
| --- | --- |
| 메뉴 이름 + 옵션 | 예) `아메리카노(ICE) (샷 추가)` — 선택된 옵션을 함께 표기 |
| 수량 | 예) `X 1`, `X 2` |
| 금액 | 해당 항목의 (단가 + 옵션가) × 수량, 예) `4,500원` |

장바구니 요약/액션:
| 요소 | 설명 |
| --- | --- |
| 총 금액 | 모든 항목의 합계 금액 표시 (예: `총 금액 12,500원`) |
| 주문하기 버튼 | 클릭 시 장바구니 내용으로 주문을 생성(전송) |

상태 처리:
- 장바구니가 비어 있을 때는 안내 문구(예: "담은 메뉴가 없습니다")를 표시하고, `주문하기` 버튼은 비활성화한다.
- 항목이 1개 이상일 때만 주문이 가능하다.

## 4. 주요 사용자 흐름 (User Flow)

1. 사용자가 주문하기 화면에 진입하면 메뉴 목록이 표시된다.
2. 원하는 메뉴 카드에서 옵션을 선택한다.
3. `담기` 버튼을 눌러 장바구니에 추가한다. (필요한 만큼 반복)
4. 장바구니에서 담긴 항목, 수량, 총 금액을 확인한다.
5. `주문하기` 버튼을 눌러 주문을 완료한다.

## 5. 금액 계산 규칙

- 항목 단가 = 메뉴 기본 가격 + 선택된 옵션들의 추가 금액 합
- 항목 금액 = 항목 단가 × 수량
- 총 금액 = 모든 장바구니 항목 금액의 합
- 모든 금액은 천 단위 콤마와 `원` 단위로 표기한다. (예: `12,500원`)

## 6. 예외 / 상태 처리

| 상황 | 처리 |
| --- | --- |
| 메뉴 데이터 로딩 중 | 로딩 표시 |
| 메뉴 조회 실패 | 오류 메시지 표시 |
| 품절 메뉴 | (관리자 재고 연동 시) 담기 버튼 비활성화 또는 품절 표시 |
| 장바구니 비어있음 | 안내 문구 표시 + 주문하기 버튼 비활성화 |
| 주문 전송 실패 | 오류 메시지 표시 후 재시도 가능 |

## 7. 비고

- 본 화면은 학습 목적이므로 사용자 인증 및 결제 기능은 포함하지 않는다.
- 메뉴는 커피 메뉴만 다룬다.
- 디자인 시안은 첨부된 와이어프레임을 기준으로 하며, 세부 스타일(색상, 폰트 등)은 추후 정의한다.
